# 충북수학체험센터 미션투어 앱 PRD (간단 버전, MVP v1)

## 1. 문서 개요
- 문서명: 충북수학체험센터 미션투어 앱 PRD
- 버전: v1.0 (Simple MVP)
- 작성일: 2026-04-20
- 플랫폼: Flutter (Android 우선)

## 2. 제품 목표
- 아주 간단한 구조로, 초등 저학년이 스토리텔링을 따라 4개 부스를 순서대로 체험하도록 만든다.
- 사용자는 각 부스에서 문제를 풀고 다음 부스로 이동한다.
- 최종적으로 4개 부스를 모두 완료하면 엔딩/완료 화면을 보여준다.

## 3. 대상 사용자
- 1차 대상: 초등 저학년
- 2차 대상(차기): 초등 고학년, 중고등

## 4. 핵심 컨셉
- 홈 화면에서 `초등 저학년` 버튼을 누르면 스토리텔링 시작
- 스토리 진행에 맞춰 아래 4개 부스를 순회
1. 수학체험실
2. 수학역사실
3. 수학놀이실
4. 수학융합실
- 각 부스에서 1개 이상 문제를 해결해야 다음 부스로 진행 가능

## 5. 사용자 플로우
1. 앱 실행
2. 첫 화면(미션 선택 화면) 진입
3. `초등 저학년` 버튼 클릭
4. 오프닝 스토리 화면
5. 부스 1 문제 풀이
6. 부스 2 문제 풀이
7. 부스 3 문제 풀이
8. 부스 4 문제 풀이
9. 최종 완료/엔딩 화면

## 6. 정보 구조(IA) 및 페이지 수
- 목표 총 페이지 수: 15~20페이지
- MVP 기준 권장: 16페이지

### 권장 16페이지 구성
1. 스플래시
2. 홈(미션 선택)
3. 초등 저학년 코스 소개
4. 스토리 오프닝
5. 부스 진행 현황(맵/순서 안내)
6. 수학체험실 소개
7. 수학체험실 문제
8. 수학역사실 소개
9. 수학역사실 문제
10. 수학놀이실 소개
11. 수학놀이실 문제
12. 수학융합실 소개
13. 수학융합실 문제
14. 중간 결과(점수/진행률)
15. 최종 완료/엔딩
16. 보상/다음 활동 안내

## 7. 기능 요구사항 (MVP)

### 7.1 홈/시작
- FR-001: 첫 화면에 코스 선택 버튼을 표시한다.
- FR-002: `초등 저학년` 버튼 클릭 시 스토리텔링이 시작된다.

### 7.2 스토리텔링
- FR-101: 오프닝 스토리 텍스트/이미지를 제공한다.
- FR-102: 스토리는 부스 진행 순서와 연결되어야 한다.

### 7.3 부스 미션
- FR-201: 4개 부스를 고정 순서로 제공한다.
- FR-202: 각 부스에는 최소 1문제 이상이 있어야 한다.
- FR-203: 정답 시 다음 부스로 이동 가능하다.
- FR-204: 오답 시 재시도 가능하다.

### 7.4 진행도/완료
- FR-301: 현재 진행 부스(1/4, 2/4 등)를 표시한다.
- FR-302: 4개 부스 완료 시 최종 완료 화면으로 이동한다.
- FR-303: 완료 화면에서 축하 메시지와 결과를 보여준다.

## 8. 비기능 요구사항
- NFR-001: 화면 전환은 사용자 체감상 즉시(1초 내) 반응한다.
- NFR-002: 저사양 안드로이드 기기에서도 기본 동작이 가능해야 한다.
- NFR-003: 네트워크 의존도를 낮춰 단순 로컬 데이터 기반으로도 동작 가능해야 한다.

## 11. 수용 기준 (Acceptance Criteria)
- AC-001: 홈에서 `초등 저학년` 버튼 클릭 시 스토리 시작 화면으로 이동한다.
- AC-002: 사용자는 4개 부스를 순서대로만 진행할 수 있다.
- AC-003: 각 부스 문제 정답 처리 후 다음 부스로 이동된다.
- AC-004: 마지막 부스 완료 후 엔딩 화면이 반드시 노출된다.
- AC-005: 전체 플로우(시작~완료)가 앱 중단 없이 동작한다.

## 12. 개발 우선순위
1. 첫 화면 레이아웃 확정(현재 이미지 스타일 반영)
2. `초등 저학년` 버튼 클릭 이벤트 연결
3. 스토리 화면 + 4개 부스 문제 화면 구성
4. 진행도 상태관리(현재 부스 인덱스)
5. 완료 화면 연결
