# 레이아웃 점검 보고서

작성일: 2026-04-25

## 점검 범위

- 홈 화면
- 챕터별 스토리 화면
- 공통 퀴즈 화면
- 챕터2 퍼즐 화면
- 챕터2 QR 인증 화면
- 챕터3 산가지/마방진 문제 화면

## 전체 판단

현재 앱은 초등학생 대상 미션형 콘텐츠라는 점에서 버튼을 크게 만들고, 캐릭터와 문제 이미지를 크게 보여주려는 방향은 적절하다. 다만 화면마다 버튼 크기, 글자 크기, 시각 자료의 비중이 다르게 적용되어 있어 사용자가 매 화면마다 새 규칙을 배우는 느낌이 있다.

특히 퀴즈 화면은 최근 수정으로 이미지 영역이 커졌지만, 하단 버튼과 테스트 버튼이 함께 남아 있어 세로 공간을 많이 차지한다. 스토리 화면은 구조가 익숙하고 이해하기 쉽지만, 말풍선과 버튼이 모두 큰 비중을 차지해 작은 화면에서는 캐릭터가 눌려 보일 수 있다.

## 화면별 분석

|화면 유형|현재 상태|익숙한 사용성 관점의 평가|
|---|---|---|
|홈 화면|카드 2열, 하단 시작 버튼|카드형 선택 구조는 익숙하다. 다만 실제 선택 가능한 카드는 하나뿐이라 나머지 카드가 눌릴 것처럼 보일 수 있다.|
|스토리 화면|상단 앱바, 배경 이미지, 캐릭터, 말풍선, 하단 다음 버튼|흐름은 명확하다. 다만 버튼과 말풍선 크기가 커서 캐릭터 중심 화면인지 대사 중심 화면인지 우선순위가 흔들릴 수 있다.|
|4지선다 문제|상단 문제, 큰 시각 자료, 1x4 보기, 제출 버튼|1x4 보기는 초등학생에게 익숙하고 안전하다. 하지만 보기 버튼의 텍스트/높이가 문제별로 다소 커서 시각 자료와 경쟁한다.|
|하노이 문제|인터랙티브 조작 영역 + 보기 버튼|직접 움직여 보는 구조는 좋다. 조작 영역과 객관식 보기가 함께 있어 문제 의도가 명확하다.|
|챕터2 퍼즐|조각 보관함, 보드, 목표 모양, 하단 조작 버튼|기능은 충분하지만 조각 보관함/목표/보드가 한 화면에서 경쟁한다. 초등 저학년에게는 '무엇을 먼저 해야 하는지'가 약간 복잡할 수 있다.|
|QR 인증|스캐너, 직접 입력, 테스트 버튼, 최근 결과|테스트 편의성은 좋다. 실제 운영 화면에서는 보조 입력/테스트 버튼이 시각적 부담이 될 수 있다.|
|마방진 문제|큰 마방진, 빈칸 탭, 숫자패드, 제출|문제 방식은 직관적이다. 다만 현재는 문제 문장, 마방진, 하단 버튼이 모두 커서 세로 스크롤이 생기기 쉽다.|

## 주요 문제점

1. 버튼 크기와 우선순위가 화면마다 다르다.

현재 일반 퀴즈의 제출/힌트 버튼은 높이 72 수준, 마방진은 56 수준, 퍼즐은 58 수준이다. 숫자 자체는 크게 문제되지 않지만 버튼 텍스트 크기와 간격까지 함께 커서, 화면 전체가 버튼 중심으로 보인다.

2. 테스트용 버튼이 실제 학습 흐름과 같은 위계로 보인다.

`테스트용: 문제 건너뛰고 다음으로`, `테스트용: 인증 건너뛰고 다음으로` 버튼이 화면 하단에 계속 노출된다. 개발 중에는 편하지만, 아이들이 보는 화면에서는 핵심 행동처럼 보일 수 있다.

3. 문제 이미지와 선택지의 관계가 아직 완전히 정돈되지 않았다.

4지선다를 1x4로 바꾼 것은 좋다. 다만 모든 보기 버튼을 넓게 펼치면 선택지는 안정적이지만, 문제 이미지 아래에 너무 큰 목록이 붙는 느낌이 있다. 보기 버튼은 더 낮고 조밀해도 충분히 누르기 쉽다.

4. 스토리 화면 컴포넌트가 여러 파일에 반복 구현되어 있다.

챕터1, 챕터2, 챕터3 스토리 화면은 거의 같은 구조를 각 파일에서 따로 구현하고 있다. 그래서 글자 크기, 캐릭터 높이, 버튼 모양, 배경 전환 규칙이 조금씩 달라질 가능성이 높다.

5. 라운드 값이 전반적으로 크다.

말풍선 `22`, 버튼 `12~16`, 카드 `14~18` 등 큰 radius가 많이 쓰인다. 어린이용 UI로는 부드러운 인상이 좋지만, 모든 요소가 둥글면 위계가 약해지고 화면이 장난감처럼 보일 수 있다.

## 수정 의견

### 1. 버튼 크기 표준화

권장 기준:

|용도|권장 높이|권장 글자 크기|
|---|---:|---:|
|주요 행동 버튼|56~60|22~26|
|보조 버튼|48~54|18~22|
|테스트/개발 버튼|40~44|14~16|
|숫자패드 버튼|52~60|24~28|

현재 `정답 제출`, `힌트`, `완성 확인`, `다음` 버튼을 이 기준에 맞추면 화면이 훨씬 안정적으로 보인다.

### 2. 테스트 버튼은 개발 모드 전용처럼 보이게 낮추기

테스트 버튼은 다음 중 하나로 정리하는 것이 좋다.

1. `kDebugMode`에서만 표시
2. 화면 우하단 작은 아이콘 버튼으로 표시
3. 길게 누르면 건너뛰는 숨은 동작으로 변경

현재처럼 전체 폭 버튼으로 두면 실제 미션보다 테스트 이동이 더 눈에 띈다.

### 3. 퀴즈 화면의 기본 레이아웃 비율 고정

권장 비율:

|영역|권장 비중|
|---|---:|
|문제 문장|10~15%|
|시각 자료|40~50%|
|보기/입력|25~35%|
|제출/힌트|10~15%|

마방진처럼 조작형 문제는 시각 자료가 화면의 절반을 차지하는 현재 방향이 맞다. 대신 하단 버튼과 테스트 버튼을 줄여야 한다.

### 4. 4지선다 버튼은 1x4 유지, 높이는 낮추기

1x4 구조는 유지하는 것이 좋다. 아이들이 위에서 아래로 읽고 하나를 고르는 방식에 익숙하다.

다만 버튼은 현재보다 낮게 잡는 편이 낫다. 보기 하나당 54~64px 정도면 충분하고, 아이콘형 도형 보기에서는 도형 자체만 크게 보이면 된다.

### 5. 스토리 화면 공통 컴포넌트화

`ChapterStoryScreen` 같은 공통 위젯을 만들고 아래 값을 파라미터로 받는 구조를 권장한다.

- 배경 이미지
- 장면 리스트
- 마지막 이동 라우트
- 캐릭터 높이 비율
- 마지막 버튼 문구

이렇게 하면 챕터별 품질이 균일해지고, 이후 챕터4를 만들 때도 훨씬 빠르게 작업할 수 있다.

### 6. 마방진 입력 UX 보완

현재 빈칸 탭 후 숫자패드가 뜨는 방식은 좋다. 추가로 아래를 권장한다.

- 선택된 빈칸에 포커스 테두리 표시
- 이미 입력된 칸을 다시 누르면 기존 숫자를 강조
- 두 칸을 모두 입력하기 전에는 제출 버튼 비활성화
- 오답 시 어떤 칸이 틀렸는지 색상으로 알려주기

## 우선순위

|우선순위|작업|이유|
|---|---|---|
|높음|테스트 버튼 축소 또는 개발 모드 전용 처리|실제 사용 흐름을 방해할 가능성이 큼|
|높음|공통 퀴즈 버튼 높이/폰트 표준화|화면 안정감에 즉시 영향|
|높음|마방진 제출 전 비활성화/포커스 표시|조작형 문제의 사용성 개선|
|중간|스토리 화면 공통 컴포넌트화|중복 감소와 장기 유지보수 개선|
|중간|챕터2 퍼즐 화면 안내 구조 정리|초등 저학년에게 복잡도가 높음|
|낮음|홈 화면 미선택 카드 비활성 스타일 적용|현재 미션 범위에서는 큰 장애는 아님|

## 결론

현재 앱은 초등학생 대상 미션 앱으로서 방향은 좋다. 큰 글자, 큰 터치 영역, 캐릭터 중심 진행은 대상 연령에 잘 맞는다.

다만 지금은 모든 요소가 동시에 크게 보이려는 경향이 있어, 화면마다 핵심 행동이 흐려진다. 앞으로는 `문제에서는 시각 자료`, `스토리에서는 캐릭터와 대사`, `조작형 문제에서는 조작판`처럼 화면별 주인공을 하나씩 정하고 나머지 UI를 한 단계 낮추는 방향이 가장 좋다.
